# university-academic-tracker – Epics

Este apartado forma parte de la documentación del proyecto **University Academic Tracker**.  
Su propósito es centralizar y describir las **épicas** que guiarán el desarrollo del sistema a lo largo de los tres cortes académicos.

---

## ¿Qué es una épica?
Una épica es una gran historia que agrupa un conjunto de funcionalidades relacionadas.  
En este proyecto, cada épica representa un bloque de trabajo principal por corte académico, y servirá como marco para organizar las historias de usuario correspondientes.

---

## Organización
- Se definirá **una épica por cada corte** (tres en total).  
- Cada épica será documentada en un archivo independiente dentro de la carpeta `epics/`.  
- Cada épica incluirá:  
  - **Nombre y descripción general.**  
  - **Objetivo:** qué busca lograr en el corte.  
  - **Justificación:** por qué es importante dentro del proyecto.  
  - **Historias de usuario asociadas (HU-#).**  
  - **Dependencias o consideraciones técnicas.**

---

## Función de las épicas en el proyecto
- Dividir el desarrollo en bloques manejables y alineados con los cortes académicos.  
- Permitir que las historias de usuario se clasifiquen y prioricen según cada corte.  
- Servir como base para planear entregables y reportes por corte.
